Building a Minecraft Server マインクラフトサーバーの構築

• This matters because when we’re building our server, we have to spec it higher due to the resource requirements of forge.

• When building a server, we need to think about what the requirements and specs are.

• サーバーを構築する際、Forgeのリソース要件のために高いスペックが必要です。

• サーバー構築時には必要な要件やスペックを考慮する必要があります。

Network ネットワーク

• Ports need to be opened in the firewall. Probably static IP or some other kind of proxy.

• Need a fast network environment. At least 30Mbit/s up, 15Mbit/s down for 8+ players, so ideally at least 100Mbit/s up and 50Mbit/s down if we’re looking for 10-20 players as a minimal requirement.

• We’ll need a network card if there’s no onboard networking (most motherboards have this). 1gb or 10gb network card should be fine if there isn’t one.

• ファイアウォールでポートを開放する必要があります。おそらく静的IPやプロキシも必要です。

• 高速なネットワーク環境が必要です。8人以上なら最低30Mbit/s上り、15Mbit/s下り、10～20人なら100Mbit/s上り、50Mbit/s下りが理想です。

• オンボードネットワークがない場合はネットワークカードが必要です。1Gbまたは10Gbのカードで十分です。

CPU CPU

• Any recent multicore cpu, either intel or amd, but amd probably has better value.

• And there’s that outside chance that we run it one of the mac studios.

• 最近のマルチコアCPU（IntelまたはAMD）が必要ですが、AMDの方がコストパフォーマンスが高いです。

• Mac Studioで運用する可能性もあります。

GPU GPU

• There’s no GPU acceleration, so GPU performance doesn’t matter (unless we run mods with some kinda GPU rendering on the server side).

• We can probably get away with the integrated graphics.

• If we use a mac studio, it’s gonna be way overspecced for the GPU.

• GPUアクセラレーションは不要なので、GPU性能は重要ではありません（サーバー側でGPUレンダリングするMODを使わない限り）。

• 内蔵グラフィックスでも十分です。

• Mac Studioを使う場合、GPUはオーバースペックになります。

Memory メモリ

• The memory requirements are higher for linux and macos, compared to windows… It shouldn’t matter, the difference should be small, 8gb for windows, 9gb for linux/macos according to the wiki.

• That’s only the memory requirements to run the server, so linux in headless mode will be the most memory efficient probably.

• In total, 16gb is a good starting point. 12gb for the server, 4gb for the OS, should be a good starting point for 10-20 players.

• The Mac studios have 32gb, so no issues there.

• LinuxやmacOSはWindowsよりメモリ要件が高いですが、差は小さいです。Windowsは8GB、Linux/macOSは9GBが目安です。

• サーバー運用にはLinuxのヘッドレスモードが最もメモリ効率が良いでしょう。

• 合計16GBが良い目安です。サーバーに12GB、OSに4GBで10～20人に対応できます。

• Mac Studioは32GBあるので問題ありません。

Storage ストレージ

• Mechanical disk is bigger than needed and slower than SSD.

• We’ll go with whatever SSD.

• For 10-20 players, a minimum of 35gb of free space is recommended.

• Given world size, backups, and OS requirements, 128gb SSD should be good enough. More doesn’t hurt.

• HDDは容量が大きいですが、SSDより遅いです。

• SSDならどれでも構いません。

• 10～20人なら最低35GBの空き容量が必要です。

• ワールドサイズやバックアップ、OS要件を考慮すると128GBのSSDが十分です。多い分には問題ありません。

Motherboard マザーボード

• Mobo depends on OS/CPU we decide to go with.

• Other notes with Motherboard. IDE express versions, lanes. These transport data from storage/gpu to main memory.

• There used to be sound cards, but these have fallen out of favor and most just have onboard sound now.

• We’ll need some kind of case and fans. Possibly a monitor, for setup.

• Would be good to just have SSH setup, so we can SSH in. The machine will also need SSH daemon and ports open for that traffic.

• マザーボードは選択するOSやCPUによって異なります。

• IDE Expressのバージョンやレーン数も重要です。これらはストレージやGPUからメインメモリへのデータ転送に関係します。

• 昔はサウンドカードが必要でしたが、今はほとんどがオンボードです。

• ケースやファン、セットアップ用のモニターも必要です。

• SSHで接続できるように設定し、SSHデーモンとポート開放も必要です。

OS OS

• Windows, Linux, Mac

• Linux or Windows would be best.

• Linux may be hard since there’s very few ppl with experience, but the same can be said for Windows and Mac too…

• Only reason Mac is even in the question is because of the expensive Mac studios that are doing nothing.

• That would be the fastest machine possible unless we pay out a lot for new hardware.

• Windows、Linux、Macが選択肢です。

• LinuxかWindowsが最適です。

• Linuxは経験者が少ないため難しいかもしれませんが、WindowsやMacも同様です。

• Macが候補に入るのは、使われていない高価なMac Studioがあるためです。

• 新しいハードウェアを購入しない限り、Mac Studioが最速のマシンです。

Java runtime environment (possibly sdk too) Javaランタイム環境（SDKも必要かも）

• OS, windows, linux, mac

• OSはWindows、Linux、Macに対応しています。

Firewall ファイアウォール

• Firewall configuration is important for security and connectivity.

• ファイアウォールの設定はセキュリティと接続性のために重要です。

Software and security updates ソフトウェアとセキュリティの更新

• Keep your server software and OS up to date for security and stability.

• サーバーソフトウェアとOSは常に最新の状態に保ち、セキュリティと安定性を確保しましょう。

Backups バックアップ

• We can probably build the mod, compile and move it to the mods directory of our forge enabled server and it should work.

• Need to test, along with backups.

• Modを作成し、コンパイルしてForge対応サーバーのmodsディレクトリに移動すれば動作するはずです。

• バックアップとともにテストが必要です。
